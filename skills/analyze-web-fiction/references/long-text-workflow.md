# 长文本分章与分块流程

1. 运行 `python scripts/split_novel.py <原文.txt> --output <目录> --chunk-chars 80000`。
2. 读取 `manifest.json`，检查章节数、标题、未匹配前缀、各块范围和超长章节标记；章节异常时用 `--chapter-pattern` 并先 `--dry-run`。
3. 同时采用两种观察：均匀位置取样识别结构演变；“目标章＋相邻章”的事件簇追踪人物关系、情感余波、价值选择和伏笔兑现。孤立章节不能支撑纵向情感或主题结论。
4. 顺序阅读全文时，每块先写不带理论术语的体验底稿，再记录信息状态、承诺与兑现、关系变化、设定新约束和价值代价。
5. 每 8–12 块做阶段归纳，至少保留两个竞争性解释。最终综合后回查原块验证关键判断和至少一个反例。

分块优先遵循章节边界。单章超长时才在段落边界切分并在 manifest 标记。不要把字符块边界误当成叙事边界，也不要把均匀抽样伪装成完整阅读。
