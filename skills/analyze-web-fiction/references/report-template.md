# 报告组织建议

模板用于保证证据与层次，不要求逐栏填满。默认把主要正文控制在约 3000–5000 个中文字符；证据较多时放入附录，避免核心判断在多个栏目反复出现。

## 分析范围与体验底稿

说明完整文本或取样范围、清洗情况与结论边界。用简短列表记录第一遍真实出现的期待、困惑、紧张、感动、好笑或疲劳及触发点，不先套术语。

## 竞争性解释

提出至少两个不同的作品模型，各用证据与反例检验。说明为何选择主解释、次解释适用于哪些阶段，以及主解释解释不了什么。不要自动把“解释权、认知爽、冲突转译”当作所有作品的默认答案。

## 核心洞见

正文只保留 3–5 个不重复的洞见。每节包含位置证据、叙事选择、读者状态变化、收益与代价；必要时给最小修改。整书报告必须覆盖但不必独立设栏：

- 设定新颖性：说明同类基线及新意是否进入剧情因果。
- 情感显微镜：分析至少一条完整微反应链与后续关系余波。
- 文以载道：展示价值命题如何进入不同人物的选择、代价和后果。
- 情绪与兑现：讨论情绪波形、爽点轮换、承诺回收与钩子疲劳。

## 有效性诊断

只指出最有效的机制、最明显的断裂和一项最高优先级修改。不要在此重述核心洞见。

## 创作手法总评

用一句话提炼作品真正独有的发动机，再给出“最值得学习的一招”和“最该警惕的一处”。除非用户要求，不打分。

## 证据附录

列出章节坐标、短引文、情绪曲线、关键反例、取样缺口和判断置信度。正文已经解释过的观点不在附录重复论证。
