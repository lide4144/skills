# 网文创作眼

把小说视为按时间展开的读者体验，而非待摘要的信息。解释作者在何处做了什么选择、该选择怎样改变读者的认知与情绪，以及换一种写法会损失什么。

## 工作流

1. 先确认分析尺度：片段/章节重在局部机制，卷/整书重在模式演变。不要用局部证据冒充全书结论。
2. **先体验，后解释**：第一遍暂不套术语，只记录何处产生期待、困惑、紧张、感动、好笑、疲劳或跳读冲动，以及触发它的文本动作。
3. 建立事件坐标：位置、局面、已知/未知信息、冲突主体、情绪压力、兑现与余波。忽略广告、网站提示和作者感言，除非它们是研究对象。
4. 阅读 [analysis-framework.md](references/analysis-framework.md)。为作品提出至少两个竞争性解释，比较各自能解释的证据、反例与盲区，再选择主解释；不要默认一切都属于“解释权/认知爽”。
5. 分析整书或深度拆书时，必须检查设定新颖性、价值承载与情感细腻度；证据不足就报告不足。所有重要判断绑定可定位证据并区分事实、强推断和弱推断。
6. 先解释“怎样生效”，再评价“是否有效”。若理论说得漂亮却解释不了第一遍的实际阅读感受，降低该理论置信度。批评要指出断裂机制并给出最小修改。
7. 根据 [report-template.md](references/report-template.md) 输出精炼正文、无术语的“读者席”、可迁移的“灵感工坊”和证据附录。读者席与灵感工坊是深度报告的必需输出，不得用编辑总评代替。

## 分析纪律

- 少复述：情节概述最多占正文的 10%，只保留理解机制必需的因果链。
- 禁止空标签：“节奏紧凑”“人物鲜明”“很爽”“有伏笔”若无机制与证据，不算分析。
- 区分事件速度与情绪节奏；事件多不等于读者情绪有起伏。
- 区分表层摩擦、资源/规则冲突与价值冲突。文本没有深层价值碰撞时，明确说没有，不强行拔高。
- 区分作者意图与文本效果。意图只能推断，使用“文本呈现出的功能是”等措辞。
- 同时寻找收益与代价。一个有效技法也可能透支悬念、人物可信度或后续升级空间。
- 对跨段、跨章结论检查前后演变，避免用开篇样本概括全书。
- 检查框架偏见：若不同作品反复得到同一结论，必须证明这是文本共性，而不是分析器只会使用同一把锤子。
- 引用原文从短，避免连续搬运正文。

## 长文本

整书分析采用“分块观察 → 模式归纳 → 反例校验 → 全书综合”。每块记录新信息、冲突、情绪波形、兑现与遗留承诺；综合时统计反复出现的模式，并主动寻找至少一个反例。取样分析必须显著标注取样范围与结论边界。

遇到超过单次上下文容量的原文，先运行 `python scripts/split_novel.py <小说文件> --output <分块目录>`。读取 `manifest.json` 检查章节识别和分块范围，再按块顺序分析。详见 [long-text-workflow.md](references/long-text-workflow.md)。`scripts/analyze_novels.py` 复用同一分章逻辑；先用 `--mode sample` 验收，再用 `--mode full` 覆盖全文。

## 输出底线

最终报告必须同时服务两种视角：

- 普通读者得到可感的阅读旅程：何处想追、何处疲劳、记住什么画面、亲近或不信任谁、下一步最想知道什么。这里禁止使用“信息差、冲突转译、情绪价值”等分析术语。
- 写作者得到可迁移的灵感：机制、成立条件、变体方向、故事问题与模仿风险。灵感必须由已分析文本生长出来，不能给通用写作鸡汤。

结尾用一句话提炼作品独有的创作发动机，再列出最值得学习的一招和最该警惕的一处。
