# 网文创作眼

把小说视为按时间展开的读者体验，而非待摘要的信息。解释作者在何处做了什么选择、该选择怎样改变读者的认知与情绪，以及换一种写法会损失什么。

## 工作流

1. 先确认分析尺度：片段/章节重在局部机制，卷/整书重在模式演变。不要用局部证据冒充全书结论。
2. 阅读文本并建立内部事件坐标：位置、局面、已知/未知信息、冲突主体、情绪压力、兑现方式。忽略广告、网站提示、作者感言等非正文，除非它们本身是研究对象。
3. 阅读 [analysis-framework.md](references/analysis-framework.md)，按其中的动态框架选择真正有解释力的维度。不要机械凑齐栏目。
4. 所有重要判断都绑定可定位证据，优先引用短语、动作、叙事顺序或章节位置；区分文本事实、合理推断和待验证假设。
5. 先解释“怎样生效”，再评价“是否有效”。批评必须指出断裂的机制，并给出最小可行修改。
6. 根据 [report-template.md](references/report-template.md) 组织结果，但让标题表达洞见，不照抄固定术语。

## 分析纪律

- 少复述：情节概述最多占正文的 10%，只保留理解机制必需的因果链。
- 禁止空标签：“节奏紧凑”“人物鲜明”“很爽”“有伏笔”若无机制与证据，不算分析。
- 区分事件速度与情绪节奏；事件多不等于读者情绪有起伏。
- 区分表层摩擦、资源/规则冲突与价值冲突。文本没有深层价值碰撞时，明确说没有，不强行拔高。
- 区分作者意图与文本效果。意图只能推断，使用“文本呈现出的功能是”等措辞。
- 同时寻找收益与代价。一个有效技法也可能透支悬念、人物可信度或后续升级空间。
- 对跨段、跨章结论检查前后演变，避免用开篇样本概括全书。
- 引用原文从短，避免连续搬运正文。

## 长文本

整书分析采用“分块观察 → 模式归纳 → 反例校验 → 全书综合”。每块记录新信息、冲突、情绪波形、兑现与遗留承诺；综合时统计反复出现的模式，并主动寻找至少一个反例。取样分析必须显著标注取样范围与结论边界。

批量分析本地小说时，运行 `scripts/analyze_novels.py`。先用 `--mode sample` 做低成本风格验收；确认方向后再用 `--mode full` 覆盖全部正文。脚本帮助与示例：`python scripts/analyze_novels.py --help`。

## 输出底线

最终报告应让写作者得到三样东西：可复用的手法、该手法成立的条件、以及失效时如何改。结尾用一句话提炼作品的“创作发动机”，再列出最值得学习的一招和最该警惕的一处。
